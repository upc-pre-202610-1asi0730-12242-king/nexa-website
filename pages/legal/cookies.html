This Cookie Notice explains the limited local browser state used by the Nexa academic website and WebApp.

Academic v1 scope No production consent manager

Nexa v1 does not run production advertising cookies or a real cookie preference platform.

01

Cookie Notice

Nexa v1 includes a static academic website. This page describes local browser state used to keep interface preferences and navigation stable while reviewing the product.

02

What cookies are

Cookies and similar browser storage technologies can remember small pieces of state in a browser. In this prototype, the relevant mechanism is primarily cookies rather than a production cookie stack.

03

What Nexa uses in this academic environment

The public website may use local browser state for UI preferences and navigation. The linked WebApp uses session state for authenticated role navigation and connected workspace flows.

04

Essential local browser state

Some local state remembers language or interface choices. Authentication and operational persistence belong to the connected WebApp services, not the public website.

05

Analytics and tracking

Nexa v1 does not run production advertising cookies. If academic analytics or real consent tooling is added in a later milestone, the notice should be updated before activation.

06

How to clear browser data

You can clear local site data from your browser settings. Removing that data may reset language preferences, login state and role navigation in the WebApp.

07

Workspace access

For questions about this academic notice, use the workspace registration flow or email the team at hello@nexa.lat.
