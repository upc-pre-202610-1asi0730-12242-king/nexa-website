01

Cookie Notice

Nexa v1 is a static academic demo. This page describes local browser state that may be used to keep interface preferences or demo navigation stable while reviewing the prototype.

02

What cookies are

Cookies and similar browser storage technologies can remember small pieces of state in a browser. In this prototype, the relevant mechanism is primarily local storage rather than a production cookie stack.

03

What Nexa uses in this academic demo

The public website may use local browser state for UI preferences and demo navigation. The linked WebApp may use local storage or session state for demo login, role navigation and frontend-only flow simulation.

04

Essential local browser state

Some local state helps the demo remember language or interface choices. This does not represent a production account system, enterprise authentication or operational data persistence.

05

Analytics and tracking

Nexa v1 does not run production advertising cookies. If academic analytics or real consent tooling is added in a later milestone, the notice should be updated before activation.

06

How to clear browser data

You can clear local site data from your browser settings. Removing that data may reset language preferences, demo login state and role navigation in the WebApp preview.
