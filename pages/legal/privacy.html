Privacy & Policy

This page explains how Nexa handles demo data, local browser state and simulated product flows in the academic prototype.

Transparent handling Demo data only

Do not enter sensitive real personal or business information into the academic prototype or Fake API demo flows.

What Nexa Is What Data May Be Used in the Demo Simulated Fake API Data Local Browser Storage Payment Method Previews AI Assistant Preview External Integrations Not Active in v1 User Rights and Contact Data Deletion Request
01

What Nexa Is

Nexa is an academic frontend prototype for B2B cold-chain operations. It presents product, inventory, dispatch, document and buyer-tracking workflows for demonstration.

02

What Data May Be Used in the Demo

The demo may include sample names, companies, SKUs, buyer requests, dispatch states, document references and operational notes. These should be synthetic or academic examples.

03

Simulated Fake API Data

The WebApp may use Fake API records to model workflows. Fake API data is not a production data store and should not be treated as verified operational truth.

04

Local Browser Storage

The public site may use local browser state for language preferences or mock session behavior. Clearing browser data may remove these local preferences.

05

Payment Method Previews

Payment-related interface elements are visual previews. Nexa v1 does not collect card details, process transactions or connect to a payment processor.

06

AI Assistant Preview

AI assistant behavior, when shown, is a prototype preview or simulated workflow. Nexa v1 does not provide a live AI service for operational decisions.

07

External Integrations Not Active in v1

GPS, IoT sensors, SUNAT, payment providers and external customer portals are not active production integrations in v1. They may appear only as roadmap or simulated scope.

08

User Rights and Contact

Anyone who has shared demo information with the team may ask what was received, request correction, or ask that demo records be removed from academic materials where practical.

09

Data Deletion Request

If you want demo data removed or purged from a review environment, contact the academic team and identify the sample record or submission you want removed.
